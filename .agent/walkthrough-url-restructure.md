# Обзор — Глобальная реструктуризация URL

Мы успешно завершили основную модернизацию структуры URL проекта. Теперь приложение использует `/dashboard` для пользовательских страниц, а `сообщения` (messages) и `избранное` (favorites) были перенесены на верхний уровень для удобства доступа и улучшения SEO.

## Выполненные изменения

### 1. Реструктуризация директорий и маршрутов
- Переименована директория `app/[locale]/(main)/profile` в `app/[locale]/(main)/dashboard`.
- Перемещены `profile/messages` в корневой маршрут `messages`.
- Перемещены `profile/favorites` в корневой маршрут `favorites`.

### 2. Конфигурация и редиректы
- Настроены постоянные 301-редиректы в `next.config.ts` для обеспечения обратной совместимости:
  - `/profile` -> `/dashboard`
  - `/profile/messages` -> `/messages`
  - `/profile/favorites` -> `/favorites`
  - Все подпути обрабатываются корректно.

### 3. Основная логика и Middleware
- Обновлен `lib/supabase/middleware.ts` для защиты пути `/dashboard` вместо `/profile`.
- Исправлен `lib/actions/listings.ts` для использования `revalidatePath('/dashboard')`.
- Обновлены хуки `use-create-listing` и `use-listing-actions` для перенаправления на новые корректные пути.

### 4. Компоненты и навигация
- Обновлены `NAV_LINKS` и все компоненты боковой панели/навигации.
- Исправлена логика условий в макетах `Header`, `Footer` и `BottomNavBar` для корректного определения страниц панели управления (dashboard).
- Исправлены ссылки в `Overview`, `UserMenu`, `Notifications` и `SellerProfile`.

### 5. База данных и тестирование
- Добавлена SQL-миграция [20260202110000_fix_message_links.sql](file:///home/creator/slovor-mp/supabase/migrations/20260202110000_fix_message_links.sql) для обновления формирования ссылок в уведомлениях в триггере БД.
- Обновлены все E2E-тесты в папке `e2e/`, соответствующие файлы тестов переименованы согласно новой терминологии (`dashboard-*`).
- Исправлены пути импорта в Storybook для всех UI-компонентов.
