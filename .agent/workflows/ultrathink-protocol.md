# ULTRATHINK PROTOCOL

**Trigger (Adaptive Mode)**:
1.  **ALWAYS ON for**: Refactoring, creating new features, fixing bugs that have failed once, architecture changes.
2.  **OPTIONAL for**: Simple typo fixes, text changes, css tweaks (to save User Credits).
3.  **Fallback**: If a "simple" task fails verify -> ESCALATE to Ultrathink immediately.

## Phase 1: Contextual Analysis

1.  **Intent Interpretation**: Decode the user's underlying goal, not just their literal request.
2.  **Cognitive Load Assessment**: Simplify the solution to minimize mental overhead for the user.
3.  **Sentiment Alignment**: Match the technical depth to the user's expertise level.

## Phase 2: Technical Impact Analysis

Evaluate the proposed solution against these dimensions:

1.  **Performance**: Critical path rendering, bundle size, network latency.
2.  **Scalability**: Coupling co-efficiency, maintenance burden (6-month horizon).
3.  **Accessibility**: WCAG AAA compliance (Contrast, ARIA, Keyboard).
4.  **Security**: Input sanitization, authentication boundaries.

## Phase 3: Solution Synthesis

1.  **Innovation Check**: Reject generic solutions if a superior, "Avant-Garde" approach exists.
2.  **Implementation Steps**:
    - **Foundation**: Types, Schemas, Core Logic.
    - **Implementation**: Components, Hooks, Services.
    - **Verification**: Automated Tests, Manual Validation.

## Phase 4: Execution & Reflection

- **Execute**: Write the code.
- **Self-Correction**: "Is this the optimal solution?" If no, refactor immediately.
