# CSP Blocking Next.js Inline Scripts

**Date:** 2026-02-22 19:05
**Files:** proxy.ts
**Severity:** high

## Problem

`BROWSER_ERROR: Executing inline script violates the following Content Security Policy directive...` appearing across all pages in the Dev Server terminal and Browser Console. The browser was blocking inline Next.js scripts (hydration, Fast Refresh, third-party) from executing.

## Root Cause

`proxy.ts` defined a strict CSP using `'nonce-...'` and `'strict-dynamic'` in the `script-src` directive. When `'strict-dynamic'` is present, modern browsers ignore the `'unsafe-inline'` fallback. Since Next.js injected inline scripts (like hydration data) do not automatically receive this nonce unless explicitly handled with `<NextScript nonce={nonce} />` (which requires passing the nonce from Middleware to Layout headers), the browser blocked all inline scripts, breaking hydration and throwing thousands of console errors.

## Solution

Removed `'nonce-${nonce}'` and `'strict-dynamic'` from the `script-src` directive in `proxy.ts`, relying on `'self'` and `'unsafe-inline'`. This is the standard, functional configuration for a Next.js app where strict nonces haven't been meticulously passed down to all generated inline scripts.

## Prevention

- Always test Content Security Policies in the browser console.
- When adding strict CSP to Next.js, ensure full integration with `<NextScript nonce={nonce} />` or stick to a more permissive CSP (`unsafe-inline`) to avoid breaking the application hydration.
- Automatically scan the browser console metrics via Playwright occasionally.
