# Fix Plan: Устранение "красных крестов" в GitHub Actions

**Problem:** GitHub Actions ран #150 горит красным, хотя все тесты и билд проходят успешно (`verify: success`). Сбой происходит на этапе `deploy` (vercel-action).
**Created:** 2026-03-21 09:45

## Analysis

- **Верификация кода**: Шаг `verify` в `ci.yml` (lint + type-check + tests + build) успешно проходит. Все 405 тестов зеленые.
- **Сбой деплоя**: Шаг `deploy` падает из-за конфликта настроек или отсутствия актуальных секретов GitHub (`VERCEL_TOKEN`, `VERCEL_PROJECT_ID`).
- **Избыточность**: В репозитории уже активна официальная интеграция Vercel GitHub, которая дублирует функции `vercel-action`. Это создает лишние проверки, которые ломают общий статус коммита.

## Fix Steps

1. [ ] **Оптимизация `ci.yml`**:
    - Удалить полностью работу (job) `deploy` из `.github/workflows/ci.yml`.
    - Оставить только работу `verify`, так как она гарантирует качество кода.
2. [ ] **Верификация**:
    - Убедиться, что после пуша в `dev` остается только одна проверка `CI Checklist`, которая будет зеленой.
3. [ ] **Рекомендация по Vercel**:
    - Если авто-деплой Vercel все еще падает, посоветовать пользователю проверить переменные окружения в панели Vercel (особенно те, что валидируются в `src/lib/env.ts`).

## Files to Modify

- `.github/workflows/ci.yml` — удаление job `deploy`.

## Risks & Considerations

- Пользователь может сознательно хотеть деплой именно через Action (редко при наличии интеграции).
- Нужно убедиться, что Vercel сам подхватывает пуши в `dev`.

## Test Coverage

- После пуша проверить статус GitHub Actions для нового коммита.
