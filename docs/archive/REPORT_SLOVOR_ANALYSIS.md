# Отчёт — аудит проекта Slovor MP

Краткое резюме
- Тип проекта: Next.js + TypeScript + React + Tailwind + Supabase
- Текущее состояние: beta / активная разработка. Проект содержит App Router (`app/`), тесты (Vitest, Playwright), Storybook и Docker конфигурации.

Основные находки (высокий приоритет)
- Секреты в репозитории: файл `.env.local` содержит Supabase service role и anon ключи — нужно немедленно удалить/регенерировать.
- В репозитории есть артефакты сборки (`.next/`) — `.gitignore` корректно их указывает, но стоит проверить, нет ли закоммиченных артефактов.

Технические рекомендации
- Безопасность: переместить секреты в CI secrets / Vault; регенерировать Supabase keys.
- CI: добавить workflow, который запускает `npm run verify` на PR (если ещё нет).
- Docker: добавлен `docker-compose.full.yml` (локальная инфраструктура: Postgres, Redis, MinIO), рекомендую использовать для локальной разработки.

Тесты
- Добавлены базовые unit-тесты для утилит: `listing-utils`, `formatting`, `cn` (Vitest). Их местоположение: `__tests__/utils/*`.

План миграции
- См. `docs/MIGRATION_NEXT15.md` — пошаговый план обновления и валидации.

Следующие шаги
1. Подтвердить удаление секретов из репозитория и регенерацию ключей.
2. Запустить CI-процедуры: `npm ci && npm run verify` и исправить найденные ошибки.
3. Прописать и протестировать docker-compose при локальной разработке.

Если нужно, я могу:
- Запустить `npm install` и `npm run verify` локально и прислать логи.
- Создать PR с исправлениями `.gitignore`, удалением `.next` и безопасностью env.
