# Структура сайта Slovor Marketplace

## 1. Введение

Данный документ описывает предлагаемую структуру сайта премиум-маркетплейса объявлений Slovor Marketplace, предназначенного для Словакии. Цель документа — предоставить четкое и полное описание навигации, меню, основных страниц и их функционала, которое может быть использовано искусственным интеллектом для реализации или доработки проекта.

## 2. Общая структура сайта

Сайт Slovor Marketplace будет иметь следующую высокоуровневую структуру, ориентированную на удобство пользователя и эффективное взаимодействие с объявлениями:

*   **Главная страница**: Точка входа, обзор популярных объявлений и категорий.
*   **Категории**: Страница для просмотра всех доступных категорий объявлений.
*   **Объявления**: Страницы для просмотра списков объявлений (поиск, фильтрация) и детального просмотра отдельных объявлений.
*   **Подача объявления**: Процесс создания и публикации нового объявления.
*   **Пользовательский профиль**: Управление учетной записью, объявлениями, избранным.
*   **Информационные страницы**: Дополнительная информация о проекте, условиях использования, политике конфиденциальности и т.д.

## 3. Навигация и меню

### 3.1. Главное меню (Шапка сайта)

Главное меню должно быть интуитивно понятным и предоставлять быстрый доступ к ключевым разделам сайта.

| Элемент меню | Ссылка / Функционал | Описание |
| :----------- | :------------------ | :--------- |
| **Логотип**  | `/` (Главная страница) | Переход на главную страницу. |
| **Поиск**    | `/listings?search={query}` | Поле для ввода поискового запроса по объявлениям. |
| **Категории**| `/categories`       | Переход на страницу со списком всех категорий. |
| **Подать объявление** | `/post` | Переход на страницу для создания нового объявления. |
| **Вход / Регистрация** | `/login` / `/register` | Вход в существующий аккаунт или создание нового. После входа меняется на "Мой профиль". |
| **Мой профиль** | `/profile` | Доступ к личному кабинету пользователя (виден после входа). |

### 3.2. Меню в футере (Подвал сайта)

Футер содержит ссылки на важные, но не первостепенные разделы, а также юридическую информацию.

| Раздел | Элемент меню | Ссылка |
| :----- | :----------- | :----- |
| **Популярные категории** | (Список популярных категорий) | `/categories/{slug}` |
| **Быстрые ссылки** | Главная | `/` |
| | Все объявления | `/listings` |
| | Подать объявление | `/post` |
| | Тренды рынка (Блог) | `/blog` |
| **Информация** | О нас | `/about` |
| | Условия использования | `/terms` |
| | Политика конфиденциальности | `/privacy` |
| | FAQ | `/faq` |

### 3.3. Меню категорий

На странице `/categories` должен быть полный список категорий, возможно, с подкатегориями. Каждая категория должна быть кликабельной и вести на страницу со списком объявлений этой категории.

| Элемент меню | Ссылка |
| :----------- | :----- |
| Электроника | `/categories/electronics` |
| Одежда | `/categories/clothing` |
| Дом и сад | `/categories/home-garden` |
| Спорт | `/categories/sports` |
| Дети и младенцы | `/categories/kids-baby` |
| Транспорт | `/categories/vehicles` |
| Недвижимость | `/categories/real-estate` |
| Услуги | `/categories/services` |
| *и другие...* | `/categories/{slug}` |

## 4. Основные страницы и их функционал

### 4.1. Главная страница (`/`)

*   **Баннер/Заголовок**: Приветствие, краткое описание маркетплейса.
*   **Поиск**: Крупное поле поиска с кнопкой, возможно, с автодополнением.
*   **Популярные запросы**: Список популярных поисковых запросов (например, iPhone, BMW, Byt, Kočík, Gauč, PS5), ведущих на страницы результатов поиска.
*   **Блок "Просмотр категорий"**: Несколько популярных категорий с иконками и ссылкой "Посмотреть все".
*   **Блок "Рекомендуемые объявления"**: Карусель или сетка из нескольких премиум-объявлений.
*   **Преимущества маркетплейса**: Раздел с описанием ключевых особенностей (Безопасность, Скорость, Бесплатно, Локальность).
*   **Призыв к действию**: Кнопка "Начать продавать сегодня!" или "Подать объявление".
*   **Футер**: Содержит ссылки на информационные страницы, категории, быстрые ссылки, форму подписки на рассылку.

### 4.2. Страница категорий (`/categories`)

*   **Заголовок**: "Просмотр категорий".
*   **Список категорий**: Полный список всех доступных категорий, каждая из которых является ссылкой на страницу объявлений этой категории. Возможно, с количеством объявлений в каждой категории.

### 4.3. Страница объявлений (список) (`/listings`, `/listings?search={query}`, `/categories/{slug}`)

*   **Заголовок**: "Все объявления" или название категории/результаты поиска.
*   **Фильтры и сортировка**: Панель с опциями для фильтрации (цена, местоположение, состояние, тип) и сортировки (по дате, по цене).
*   **Список объявлений**: Сетка или список карточек объявлений, каждая из которых содержит:
    *   Изображение.
    *   Заголовок.
    *   Цена.
    *   Местоположение.
    *   Краткое описание.
    *   Дата публикации.
*   **Пагинация**: Для навигации по страницам с объявлениями.

### 4.4. Страница отдельного объявления (`/listings/{id}`)

*   **Галерея изображений**: Основное изображение и миниатюры.
*   **Заголовок объявления**.
*   **Цена**.
*   **Местоположение**.
*   **Описание**: Полное описание товара/услуги.
*   **Информация о продавце**: Имя пользователя, рейтинг (если есть), ссылка на профиль продавца.
*   **Кнопки действия**: "Связаться с продавцом" (чат/телефон), "Добавить в избранное", "Пожаловаться".
*   **Похожие объявления**: Блок с рекомендациями.

### 4.5. Страница подачи объявления (`/post`)

*   **Форма**: Поля для ввода информации об объявлении:
    *   Заголовок.
    *   Описание.
    *   Категория (выпадающий список).
    *   Цена.
    *   Местоположение.
    *   Изображения (загрузка нескольких файлов).
    *   Контактная информация.
*   **Кнопка "Опубликовать объявление"**.
*   **Требования к авторизации**: Доступна только для авторизованных пользователей. Если пользователь не авторизован, перенаправление на страницу входа/регистрации.

### 4.6. Страницы пользователя (`/profile`, `/profile/my-listings`, `/profile/favorites`, `/login`, `/register`)

*   **Вход (`/login`)**: Форма для ввода email/пароля, ссылки на регистрацию и восстановление пароля.
*   **Регистрация (`/register`)**: Форма для создания аккаунта (email, пароль, подтверждение пароля, имя пользователя).
*   **Профиль пользователя (`/profile`)**: Основная информация о пользователе, возможность редактирования.
*   **Мои объявления (`/profile/my-listings`)**: Список объявлений, опубликованных пользователем, с возможностью редактирования, удаления, поднятия.
*   **Избранное (`/profile/favorites`)**: Список объявлений, добавленных пользователем в избранное.

### 4.7. Информационные страницы (`/about`, `/terms`, `/privacy`, `/faq`, `/blog`)

*   **О нас (`/about`)**: Информация о проекте Slovor Marketplace, его миссии и целях.
*   **Условия использования (`/terms`)**: Юридические условия использования платформы.
*   **Политика конфиденциальности (`/privacy`)**: Описание сбора и использования пользовательских данных.
*   **FAQ (`/faq`)**: Часто задаваемые вопросы и ответы.
*   **Блог (`/blog`)**: Статьи, новости, тренды рынка.

## 5. Пользовательские потоки (User Flows)

### 5.1. Поиск объявления

1.  Пользователь заходит на главную страницу.
2.  Вводит запрос в строку поиска или выбирает категорию.
3.  Переходит на страницу списка объявлений.
4.  Применяет фильтры и сортировку для уточнения результатов.
5.  Просматривает список объявлений.

### 5.2. Просмотр объявления

1.  Пользователь нажимает на карточку объявления в списке.
2.  Переходит на страницу детального просмотра объявления.
3.  Изучает информацию, изображения.
4.  Может связаться с продавцом или добавить объявление в избранное.

### 5.3. Подача объявления

1.  Пользователь нажимает кнопку "Подать объявление".
2.  Если не авторизован, перенаправляется на страницу входа/регистрации.
3.  После авторизации заполняет форму объявления.
4.  Загружает изображения.
5.  Нажимает "Опубликовать".
6.  Получает подтверждение об успешной публикации.

### 5.4. Регистрация/Вход

1.  Пользователь нажимает "Вход" или "Регистрация".
2.  Заполняет соответствующую форму.
3.  После успешной операции перенаправляется в личный кабинет или на главную страницу.

### 5.5. Управление объявлениями

1.  Авторизованный пользователь переходит в "Мой профиль" -> "Мои объявления".
2.  Просматривает список своих объявлений.
3.  Может редактировать, удалять или поднимать объявления.

## 6. Технические аспекты (без кода)

### 6.1. Роутинг

*   **Чистые URL**: Использование человекопонятных URL-адресов (например, `/listings/electronics/iphone-x`).
*   **Динамические маршруты**: Для страниц объявлений (`/listings/{id}`) и категорий (`/categories/{slug}`).
*   **Защищенные маршруты**: Для страниц пользователя (`/profile/*`, `/post`), требующих авторизации.

### 6.2. Управление состоянием

*   **Глобальное состояние**: Для данных пользователя (авторизация), корзины (если применимо), фильтров поиска.
*   **Локальное состояние**: Для компонентов (например, состояние формы подачи объявления).

### 6.3. Взаимодействие с API

*   **RESTful API**: Для получения, создания, обновления и удаления объявлений, категорий, данных пользователя.
*   **Аутентификация**: Использование токенов (например, JWT) для защиты пользовательских данных и операций.
*   **Валидация данных**: На стороне сервера для всех входящих данных.

## 7. Рекомендации для ИИ

При реализации данной структуры, ИИ должен учитывать следующие моменты:

*   **Модульность**: Разделение функционала на независимые, переиспользуемые компоненты.
*   **Адаптивность**: Дизайн должен быть адаптивным для корректного отображения на различных устройствах (десктоп, планшет, мобильный телефон).
*   **Производительность**: Оптимизация загрузки страниц, изображений и запросов к API.
*   **SEO-оптимизация**: Использование семантической разметки, мета-тегов, чистых URL для лучшей индексации поисковыми системами.
*   **Безопасность**: Реализация всех необходимых мер безопасности для защиты пользовательских данных и предотвращения мошенничества.
*   **Расширяемость**: Архитектура должна позволять легко добавлять новый функционал и разделы в будущем.

Эта структура является основой для разработки Slovor Marketplace. ИИ может использовать ее для генерации кода, создания макетов и дальнейшего развития проекта, следуя указанным принципам.
