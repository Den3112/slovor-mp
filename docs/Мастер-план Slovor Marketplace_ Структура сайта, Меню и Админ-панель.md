# Мастер-план Slovor Marketplace: Структура сайта, Меню и Админ-панель

Этот документ представляет собой финальную, детализированную спецификацию для ИИ-разработчика, охватывающую полную структуру сайта (Sitemap), навигацию, а также функционал Личного кабинета и Админ-панели.

## 1. Структура сайта (Sitemap)

Структура сайта разработана с учетом принципов **"Premium Minimalist Navigation"** и включает все необходимые разделы для доверительной и функциональной платформы.

### 1.1. Основная навигация (Header / Mobile Tab Bar)

| Название | URL | Описание | Приоритет | Примечание |
| :--- | :--- | :--- | :--- | :--- |
| **Главная** | `/` | Точка входа, витрина премиум-объявлений. | Высокий | |
| **Категории** | `/listings` | Обзор всех категорий (через Мега-меню на Desktop). | Высокий | На Mobile Tab Bar - иконка "Обзор". |
| **Подать объявление** | `/post` | Форма для создания нового объявления. | Высокий | Центральная кнопка на Mobile Tab Bar. |
| **Сообщения** | `/messages` | Внутренний чат и Inbox. | Высокий | С индикатором непрочитанных сообщений. |
| **Избранное** | `/favorites` | Сохраненные объявления. | Средний | Доступно только авторизованным. |
| **Профиль / ЛК** | `/profile` | Личный кабинет пользователя. | Высокий | Иконка Аватара. |

## 2. Структура и функционал Личного Кабинета (ЛК)

Личный кабинет (Dashboard) должен быть центром управления для пользователя, предоставляя полный контроль над объявлениями, профилем и коммуникацией.

**URL**: `/profile`

### 2.1. Обзор (Dashboard)

**Цель**: Краткий обзор ключевых метрик и быстрый доступ к основным действиям.

| Блок | Функционал | Примечание |
| :--- | :--- | :--- |
| **Профиль** | Аватар, имя, статус верификации. | Кнопка "Редактировать профиль". |
| **Статистика** | Общее количество просмотров, количество активных объявлений, средний рейтинг. | Виджеты с ключевыми KPI. |
| **Последние действия** | Последние 3 сообщения, 3 новых отзыва, 3 последних просмотренных объявления. | |
| **Быстрые ссылки** | "Подать объявление", "Продлить объявление", "Пополнить баланс". | |

### 2.2. Мои объявления (`/profile/listings`)

**Цель**: Управление всеми объявлениями пользователя.

| Функционал | Описание |
| :--- | :--- |
| **Фильтрация** | По статусу (Активно, На модерации, Истек срок, Продано, Черновик). |
| **Действия** | Редактировать, Продлить (платно), Продвинуть (платно), Снять с публикации, Удалить. |
| **Статистика** | Отображение просмотров, добавлений в избранное и количества сообщений по каждому объявлению. |
| **Массовые действия** | Возможность выбрать несколько объявлений и выполнить одно действие (например, снять с публикации). |

### 2.3. Отзывы и Рейтинг (`/profile/reviews`)

**Цель**: Управление репутацией.

| Функционал | Описание |
| :--- | :--- |
| **Полученные отзывы** | Список всех отзывов с возможностью ответить на каждый. |
| **Оставленные отзывы** | Список отзывов, которые пользователь оставил другим. |
| **Сводка рейтинга** | График распределения оценок (5 звезд, 4 звезды и т.д.). |

### 2.4. Верификация (`/profile/verification`)

**Цель**: Повышение доверия через подтверждение личности.

| Функционал | Описание |
| :--- | :--- |
| **Статус** | Отображение текущего уровня верификации (Email, Телефон, Документы). |
| **Форма** | Пошаговая форма для загрузки документов (паспорт, водительское удостоверение) с четкими инструкциями. |
| **История** | Статус запроса (На рассмотрении, Одобрено, Отклонено). |

### 2.5. Платежи и Подписки (`/profile/payments`)

**Цель**: Управление финансовыми операциями.

| Функционал | Описание |
| :--- | :--- |
| **История транзакций** | Детализация всех платежей (за продвижение, за подписки) с датой, суммой и статусом. |
| **Продвижение** | Выбор и оплата услуг по продвижению (ТОП-объявление, Выделение цветом). |
| **Настройки оплаты** | Привязка платежных карт. |

## 3. Структура и функционал Админ-панели (Admin Panel)

Админ-панель (доступна по `/admin` только для пользователей с `role: 'admin'`) — это инструмент для модерации, управления контентом и аналитики.

### 3.1. Обзор (Admin Dashboard)

**Цель**: Ключевые метрики для контроля здоровья платформы.

| Метрика | Описание |
| :--- | :--- |
| **Новые объявления** | Количество объявлений, ожидающих модерации. |
| **Активные пользователи** | Количество активных пользователей за последние 24 часа. |
| **Доход** | Общий доход за месяц/неделю (от продвижения). |
| **Статистика** | Графики роста пользователей и объявлений. |

### 3.2. Модерация объявлений (`/admin/listings`)

**Цель**: Быстрая и эффективная модерация контента.

| Функционал | Описание |
| :--- | :--- |
| **Фильтр** | По статусу (На модерации, Отклонено, Активно), по категории, по ключевому слову. |
| **Действия** | Одобрить, Отклонить (с указанием причины), Снять с публикации, Редактировать. |
| **Просмотр** | Детальный просмотр объявления с возможностью увидеть данные продавца. |

### 3.3. Управление пользователями (`/admin/users`)

**Цель**: Полный контроль над пользовательской базой.

| Функционал | Описание |
| :--- | :--- |
| **Поиск** | Поиск по Email, ID, имени. |
| **Действия** | Блокировать/Разблокировать, Сбросить пароль, Изменить роль (user/admin), Просмотр истории активности. |
| **Верификация** | Отдельный раздел для рассмотрения запросов на верификацию документов. |

### 3.4. Управление контентом (`/admin/content`)

**Цель**: Управление статическим контентом и категориями.

| Функционал | Описание |
| :--- | :--- |
| **Категории** | Создание, редактирование, удаление категорий и подкатегорий. Управление динамическими атрибутами для каждой категории. |
| **Блог** | CRUD-операции для статей в блоге. |
| **Страницы** | Редактирование статических страниц (О нас, FAQ, Условия). |

## 4. Инструкции для ИИ-разработчика (Technical Specifications)

ИИ-бот должен использовать следующие инструкции для реализации вышеописанной структуры.

### 4.1. Общие принципы реализации

*   **Стек**: Next.js (App Router), TypeScript, Tailwind CSS, Supabase (PostgreSQL, Realtime, Auth).
*   **Модульность**: Каждый блок (например, "Статистика" в ЛК, "Карточка пользователя" в Админ-панели) должен быть отдельным, переиспользуемым компонентом.
*   **Авторизация**: Использовать `middleware.ts` для защиты всех маршрутов `/profile/*` и `/admin/*`. Маршруты `/admin/*` должны быть доступны только пользователям с `role: 'admin'`.

### 4.2. Реализация Sitemap и Навигации

1.  **Header (Desktop)**: Модифицировать `/components/layout/header.tsx`.
    *   Интегрировать Мега-меню для `/listings` (см. предыдущее ТЗ).
    *   Добавить ссылки на `/blog` и `/help`.
    *   Кнопка "Подать объявление" должна быть выделена (например, `variant="default"`).
2.  **Mobile Tab Bar**: Модифицировать `/components/layout/bottom-nav-bar.tsx`.
    *   Обеспечить наличие 5 ключевых иконок: Главная, Обзор (Категории), Подать объявление (Центральная), Сообщения, Профиль.
    *   Использовать `motion.div` для плавного скрытия при скролле вниз и появления при скролле вверх.
3.  **Футер**: Создать `/components/layout/footer.tsx`.
    *   Разбить на 4 колонки, как описано в разделе 1.2.
    *   Использовать компонент `Link` из Next.js для всех внутренних ссылок.

### 4.3. Реализация Личного Кабинета (ЛК)

1.  **Layout ЛК**: Создать `/app/profile/layout.tsx`.
    *   Должен включать боковую навигационную панель (Sidebar) со всеми ссылками из раздела 1.3.
    *   Sidebar должен быть адаптивным: на десктопе — фиксированный, на мобильном — скрываться в Drawer или быть доступен через кнопку.
2.  **Dashboard**: Создать `/app/profile/page.tsx`.
    *   Использовать `useSWR` или Next.js Server Components для получения данных о пользователе и статистике.
    *   Компоненты статистики должны быть визуально привлекательными (например, с использованием `recharts` или `nivo` для графиков).
3.  **Мои объявления**: Создать `/app/profile/listings/page.tsx`.
    *   Использовать компонент `DataTable` (или аналог) для отображения списка объявлений.
    *   Для действий (Продвинуть, Продлить) использовать модальные окна с подтверждением и интеграцией с платежным API.

### 4.4. Реализация Админ-панели

1.  **Layout Админ-панели**: Создать `/app/admin/layout.tsx`.
    *   Отдельный, более строгий дизайн, отличающийся от основного сайта.
    *   Проверка роли пользователя: если `user.role !== 'admin'`, перенаправлять на `/`.
2.  **Модерация объявлений**: Создать `/app/admin/listings/page.tsx`.
    *   Использовать мощный компонент `DataTable` с возможностью поиска, сортировки и пагинации.
    *   Кнопки действий (Одобрить/Отклонить) должны быть мгновенными (оптимистичное обновление UI) с последующей проверкой ответа сервера.
3.  **Управление категориями**: Создать `/app/admin/content/categories/page.tsx`.
    *   Интерфейс для создания иерархии (дерево категорий).
    *   Форма для добавления/редактирования **динамических атрибутов** (например, для категории "Автомобили" — "Пробег", "Год выпуска").

**Приоритеты для ИИ-бота**:

| Приоритет | Задача |
| :--- | :--- |
| **Высокий** | Реализация защищенного Layout ЛК и Админ-панели. |
| **Высокий** | Реализация Sidebar и всех ссылок ЛК. |
| **Средний** | Реализация страниц Мои объявления и Обзор ЛК. |
| **Средний** | Реализация Модерации объявлений в Админ-панели. |
| **Низкий** | Реализация страниц Блог и Карьера. |
| **Низкий** | Полная реализация Управления контентом (динамические атрибуты). |
